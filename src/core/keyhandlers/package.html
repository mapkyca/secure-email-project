A package containing classes for managing local keys and connecting to remote key servers.

All classes are derived from the abstract class {@link core.keyhandlers.KeyHandler} that defines a common interface for fetching keys from different sources - files on disk, remote servers etc...

From this class, two more abstract classes are defined; {@link core.keyhandlers.KeyFile} and {@link core.keyhandlers.KeyServer} which define extra functionality for key files and key servers respectively.

Steps to add a new key handler:

  1. Create the key handler class & implement the required methods.
  2. Modify {@link core.keyhandlers.KeyHandler#loadKeysourceList(ConfigurationData, String) } to support the necessary type if necessary (don't forget to update app/doc-files/SystemConfigurationSettings.html if necessary). This should only be necessary if the class isn't instanciated with the standard key file or key server constructor format.
  3. Modify the key management UI routines.
  4. Add the appropriate entries in the .lst files so that the new key handler will appear in the UI list.
